# 前端文案中文化设计

## 目标

将静态前端页面中的所有用户可见英文文案改为中文，保证首页上传流程、结果展示区、错误提示和折叠面板文案一致，避免页面中英混杂。

## 范围

- 仅修改 `src/main/resources/static/index.html`
- 覆盖静态 HTML 文案和内联 JavaScript 生成的动态文案
- 不修改后端接口、返回字段、请求路径和页面结构

## 方案

采用最小改动方案，在现有单页结构中直接替换用户可见文案：

- 页面标题、主标题、副说明改为中文
- 上传区标题、按钮、状态提示改为中文
- 人物、图片匹配、公开账号、相关文章、调试面板等分区标题改为中文
- 空态、加载态、失败态、部分成功提示改为中文
- 动态渲染时的占位词、点击提示、来源链接提示改为中文

## 风险与控制

- 保留现有 DOM 结构和 JS 逻辑，降低样式与交互回归风险
- 不修改接口字段名，避免影响后端联调
- 保留 `Face2Info` 与 `Serper` 等专有名词，仅翻译说明性文本

## 验证

- 检查页面中不再出现面向用户的英文提示
- 确认上传前、加载中、成功、部分成功、失败场景文案均为中文
- 确认折叠面板展开/收起提示改为中文
