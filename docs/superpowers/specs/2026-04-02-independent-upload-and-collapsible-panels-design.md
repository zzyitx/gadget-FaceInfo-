# 独立上传区与折叠信息模块设计

## 目标

将首页顶部标题区与上传表单解耦，形成“顶部标题独立展示 + 左侧独立上传区 + 右侧结果区”的页面结构；同时将《相关文章》《社交账号》《调试信息》拆成 3 个独立模块，并全部默认折叠。

## 结构设计

- 顶部 `hero` 仅保留品牌标题、说明和关键指标。
- `hero` 下方新增双栏主布局：
  - 左侧为独立上传区，包含文件选择、图片预览、提交按钮、状态提示。
  - 右侧为结果区，保留人物简介、图片匹配和其余信息模块。
- 移动端回退为单列顺序：顶部标题、上传区、结果区。

## 结果区设计

- `人物简介`：常显主卡片，不折叠。
- `图片匹配`：常显辅助卡片，不折叠。
- `相关文章`：独立折叠模块，默认关闭，内部包含：
  - `相关网站来源`：读取 `image_matches`
  - `相关新闻`：读取 `news`
- `社交账号`：独立折叠模块，默认关闭。
- `调试信息`：独立折叠模块，默认关闭。

## 交互与状态

- 上传提交逻辑和接口地址保持不变。
- 左侧上传区负责上传前后状态反馈。
- 右侧模块在成功、空态、失败三种状态下都保持独立结构，不合并到同一卡片中。
- 折叠模块统一使用原生 `details/summary`，默认不添加 `open` 属性。

## 测试策略

- 静态页测试验证独立上传区容器存在。
- 静态页测试验证《相关文章》《社交账号》《调试信息》都为独立折叠模块。
- 静态页测试验证《相关文章》脚本同时引用 `image_matches` 与 `news`。
