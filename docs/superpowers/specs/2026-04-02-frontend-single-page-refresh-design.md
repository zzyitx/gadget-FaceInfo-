# 前端单页重构设计

**日期：** 2026-04-02

**目标：** 重新优化静态前端页面，删除无效的“识别线索”板块，保留独立“图片匹配”板块，并在人物简介主卡中移除维基百科与官网信息，使页面更聚焦聚合结论与辅助信息。

## 背景

当前前端页面集中在 [index.html](/D:/ideaProject/gadget/src/main/resources/static/index.html)，采用单文件 HTML、CSS、JavaScript 结构。现有问题主要有：

- 上传区被固定在左侧侧栏，首屏焦点分散。
- “识别线索”板块展示了正文证据链接、流程摘要等低价值信息，实际使用中帮助不大。
- 人物简介主卡同时展示维基百科和官网，弱化了真正应该优先突出的聚合简介。
- 结果区层级不够明确，人物简介、图片匹配、社交账号、新闻之间的主次关系不够清晰。

## 设计目标

### 1. 信息层级

页面需要明确遵循“主结论优先，辅助比对次之，扩展信息最后”的顺序：

1. 上传与状态反馈
2. 人物简介主卡
3. 图片匹配辅助比对
4. 社交账号与近期新闻
5. 调试 JSON

### 2. 视觉方向

页面采用“编辑感工具页”风格，而不是常见后台面板风格：

- 保留暖色基底，但减少堆叠式说明文案。
- 顶部区域更像产品首页首屏，上传入口直接前置。
- 主结果卡强调留白、标题层级和结论性表达。
- 次级卡片风格统一，避免每块内容都在抢视觉焦点。

### 3. 内容策略

- 删除“识别线索”整块展示与对应渲染逻辑。
- 人物简介只展示对用户有用的主结论内容，不展示 `wikipedia` 和 `official_website`。
- “图片匹配”继续独立展示，但文案语义调整为辅助比对，不再暗示它是最终结论。
- 调试 JSON 继续保留，但视觉降级到底部折叠区域。

## 页面结构

## 顶部 Hero

顶部 Hero 改为双栏布局：

- 左侧展示产品标题、简短介绍、能力标签。
- 右侧内嵌上传表单、图片预览、提交按钮、状态反馈。

这样可以让用户在首屏直接完成上传，同时保留产品定位说明。

## 主结果区

上传成功后，第一块结果固定为“人物简介”主卡，展示：

- 人物姓名
- 聚合状态标签
- 人物简介
- 可选标签列表（若后端返回）

不再展示：

- 维基百科链接
- 官网链接

## 辅助比对区

“图片匹配”保留独立板块，放在人物简介之后。其定位改为辅助比对，展示：

- 匹配图片预览
- 来源站点
- 相似度
- 来源链接

空状态与失败状态都需要强调“这是辅助信息”，避免误导为主结论。

## 扩展信息区

“社交账号”和“近期新闻”作为扩展信息，采用统一卡片风格：

- 桌面端可采用两列并排。
- 窄屏下改为单列纵向。

## 调试区

调试 JSON 放在页面底部，默认折叠，不参与主视觉竞争。

## 交互设计

### 状态反馈

状态文案收敛为五种：

- 等待上传
- 分析中
- 已完成
- 部分完成
- 请求失败

每种状态保留一行简洁说明，不再显示冗长流程描述。

### 加载体验

提交请求后，结果区先进入统一占位态，再在响应到达后刷新各模块内容。虽然仍是单页原生 JavaScript 实现，但视觉上应减少大面积跳变。

### 文案语气

整体文案采用更短、更直接的说明方式：

- 减少解释后端内部流程
- 强调“聚合结果”和“辅助比对”
- 明确区分主信息与补充信息

## 实现范围

本次仅修改静态前端 [index.html](/D:/ideaProject/gadget/src/main/resources/static/index.html)：

- 重写页面布局与样式
- 删除“识别线索”相关 DOM 与渲染函数
- 调整人物简介渲染逻辑
- 保留现有接口调用方式 `POST /api/face2info`
- 保留调试 JSON 输出能力

本次不改动后端接口结构，不新增前端构建工具，不拆分为多文件前端工程。

## 风险与兼容性

- 由于页面仍为单文件实现，结构重排需要谨慎处理 DOM 查询和状态更新，避免出现元素 ID 与渲染逻辑不匹配。
- 需要保留现有接口字段兼容性，不能因为前端文案或布局调整而假设后端一定返回新增字段。
- 若 `person.tags` 不存在，人物简介卡必须正常降级显示。

## 验证要求

改动完成后至少验证：

- 首页在桌面与移动宽度下布局正常
- 上传图片预览正常
- 成功返回时人物简介、图片匹配、社交账号、新闻都能正确渲染
- `partial` 状态下页面文案正确
- 失败状态下结果区正确降级
- 调试 JSON 仍可查看

## 非目标

本次不包含以下内容：

- 后端接口字段调整
- 引入前端框架或构建系统
- 调整 API 地址、上传协议或鉴权机制
- 新增识别证据可视化能力
